# 운동 영상 업로드 자세 분석 서비스

업로드한 운동 영상을 **YOLOv5 + MediaPipe + Random Forest** 파이프라인으로 분석해 자세 점수와 한국어 피드백을 제공하는 Streamlit 앱입니다. 기존 `Streamlit.py`(웹캠 실시간용)는 그대로 두고 별도의 `Streamlit_Upload.py`로 동작합니다.

---

## 1. 실행 방법

### 사전 조건
- Windows 11, Python 3.10.11 가상환경 (`venv/`) 활성화
- `requirements_clean.txt` 기준으로 패키지 설치 완료
- `models/` 폴더 내 `best_big_bounding.pt` 및 `benchpress/squat/deadlift` 하위 `.pkl` 파일 존재

### 실행 명령어
프로젝트 루트에서 실행합니다.

```bash
# venv의 python으로 직접 실행
.\venv\Scripts\python.exe -m streamlit run Streamlit_Upload.py
```

서버가 기동되면 브라우저가 자동으로 열리며, 수동 접속은 `http://localhost:8501` 입니다.

첫 실행 시에는 `torch.hub`가 `ultralytics/yolov5` 레포지토리를 로컬 캐시(`~/.cache/torch/hub`)에 다운로드하기 때문에 수 분이 걸릴 수 있습니다. 인터넷 연결이 필요합니다. 두 번째 실행부터는 캐시를 재사용하므로 빠르게 올라옵니다.

### 종료
터미널에서 `Ctrl + C` 를 누르면 서버가 종료됩니다.

## 2. 영상 촬영 가이드

정확한 판정을 위해 운동별 권장 각도에서 촬영해주세요. 잘못된 각도로 찍으면 MediaPipe 랜드마크가 불안정해져 점수가 낮게 나올 수 있습니다.

### 공통 권장 사항
- 한 명만 찍을 것 (YOLOv5가 가장 높은 신뢰도 한 명만 사용)
- 전신이 프레임에 모두 들어올 것 (머리~발끝)
- 조명 충분, 배경 단순하게
- 바벨/원판이 랜드마크를 너무 많이 가리지 않도록
- 영상 길이는 30초 ~ 2분 권장 (너무 길면 CPU 분석 시간이 오래 걸림)

### 벤치프레스 (정면)
- 카메라를 벤치 **발 아래쪽**에 두고 바벨이 정면으로 보이도록 배치
- 그립 너비, 허리 아치 판정에 유리
- 눈높이보다 살짝 낮게 찍는 편이 좋음

### 스쿼트 (측면)
- 정확히 옆면을 찍어 **무릎과 허리 라인**이 한눈에 보이도록 배치
- 무릎 안쪽 꺾임(`caved_in_knees`), 척추 각도, 발 간격 판정에 사용
- 대퇴골이 지면과 평행해지는 깊이까지 들어가는 구간을 포함할 것

### 데드리프트 (측면)
- 옆면에서 **바·허리·무릎**이 한 라인에 보이도록 배치
- 척추 중립(`spine_neutral`) 여부, 그립 너비 판정에 사용
- 탑 포지션(락아웃)과 바텀 포지션이 모두 영상에 들어가도록

## 3. 화면 사용법

### 사이드바 설정
- **프레임 건너뛰기**: 기본 3 프레임마다 1번 분석. 값이 클수록 분석 속도는 빨라지지만 세밀한 피드백은 줄어듭니다. CPU 환경 기준 30초 영상 기준 권장값은 3~5.
- **YOLO 사람 검출 신뢰도**: 기본 0.5. 영상에 사람이 뚜렷이 찍혀 있으면 0.5~0.7, 어둡거나 가려져 있으면 0.3~0.4 로 낮춰볼 수 있습니다.

### 분석 흐름
1. 운동 종류 선택 → 촬영 각도 안내 확인
2. 영상 파일 업로드 (mp4 / mov / avi, 용량 큰 경우 수초 대기)
3. **분석 시작** 버튼 클릭
4. 진행률 바 확인 (프레임 처리 중 현재 / 전체)
5. 완료 후 결과 화면 자동 표시

## 4. 결과 해석

### 전체 점수
```
점수 = (정상 자세로 판정된 프레임 수 / 분석된 프레임 수) × 100
```
- **80점 이상**: 전반적으로 올바른 자세. 가끔 흔들림 정도.
- **50~80점**: 일부 구간에서 특정 오류가 반복됨. 피드백 메시지에서 나온 오류 유형 집중 개선.
- **50점 미만**: 자세 교정 필요. 하단의 오류 유형별 빈도를 보고 가장 많이 나온 항목부터 수정할 것.
- **0점 또는 분석 프레임 0**: 영상에서 사람을 제대로 잡지 못했거나, 촬영 각도가 학습 데이터와 크게 달라 분류기가 작동하지 못한 상태. 촬영 가이드대로 다시 찍어보세요.

### 감지된 rep 수
분류기의 `_down → _up` 상태 전이 횟수를 세어 출력합니다. 실제 반복 수와 다소 차이가 날 수 있습니다(아래 한계 참고).

### 자세별 피드백
오류로 분류된 프레임이 많은 순서대로 한국어 피드백 메시지가 출력됩니다. 각 항목 앞에 `[N회 감지]` 표기가 붙습니다.

| 오류 키 | 운동 | 메시지 요약 |
|---|---|---|
| `excessive_arch` | 벤치프레스 | 허리 과도한 아치 → 가슴 펴고 복부 긴장 |
| `arms_spread` | 벤치/데드 | 그립 너무 넓음 → 어깨 너비 약간 넓게 |
| `arms_narrow` | 데드 | 그립 너무 좁음 → 어깨 너비보다 조금 넓게 |
| `spine_neutral` | 스쿼트/데드 | 척추 굽음 → 가슴 들고 어깨 뒤로 |
| `caved_in_knees` | 스쿼트 | 무릎 안쪽 꺾임 → 엉덩이 뒤로 빼서 일직선 |
| `feet_spread` | 스쿼트 | 발 너무 넓게 → 어깨 너비로 |

### 오류 유형별 발생 빈도 (막대 그래프)
가장 빈번히 발생한 오류를 한눈에 비교할 수 있습니다. 최다 오류부터 차근차근 교정하는 것이 효과적입니다.

### 전체 클래스 분포 (expander)
분류기가 출력한 원시 클래스 이름과 각 클래스 프레임 수를 확인할 수 있습니다. 개발/디버깅 목적 또는 자세가 어떤 구간에서 주로 판정되었는지 상세 확인용.

## 5. 알려진 한계

1. **촬영 각도 민감도**: 분류기가 특정 각도(벤치=정면, 스쿼트·데드=측면)의 데이터로 학습되었기 때문에 각도가 크게 벗어나면 정확도가 급격히 떨어집니다.
2. **한 명만 분석**: YOLOv5가 가장 높은 신뢰도로 잡은 한 명만 추적합니다. 프레임 안에 여러 사람이 있으면 의도한 대상이 아닐 수 있습니다.
3. **rep 카운트 오차**: 분류기의 `up`/`down` 라벨이 순간적으로 잘못 찍히면 카운트가 실제와 달라질 수 있습니다. 너무 빠른 템포나 파셜 rep에서 특히 그렇습니다.
4. **분석 속도 (CPU)**: GPU를 사용하지 않아 30초 영상 기준 수십 초~1분 내외가 걸립니다. 더 긴 영상은 프레임 건너뛰기 값을 5 이상으로 키워서 속도를 확보하세요.
5. **바벨·원판 가림**: 파워리프팅 특성상 바벨이 상체 랜드마크 일부를 가릴 수 있습니다. 완전히 가리는 구간은 분석에서 제외됩니다.
6. **영상 해상도**: 720p 이하 저해상도 영상은 랜드마크 정확도가 떨어질 수 있습니다. 1080p 이상 권장.
7. **짧은 영상**: 1 rep도 포함되지 않는 짧은 영상은 rep 카운트가 0으로 나옵니다. 점수는 계산되지만 통계적 의미는 약합니다.

## 6. 문제 해결

| 증상 | 원인 / 대처 |
|---|---|
| `YOLOv5 모델을 로딩하는 중입니다...` 에서 수 분째 멈춤 | 첫 실행 시 torch.hub가 github에서 리포 다운로드 중. 인터넷 속도·연결 확인. |
| `분석된 프레임: 0 / N` | YOLO가 사람을 못 찾음. 사이드바에서 YOLO 신뢰도를 0.3으로 낮추거나 영상 재촬영. |
| 점수가 비정상적으로 낮음 | 촬영 각도가 학습 시점과 크게 다를 가능성. 위 2번 촬영 가이드 재확인. |
| 분석 중 에러 / 스택트레이스 | 업로드 영상 코덱 문제일 수 있음. mp4(H.264)로 변환 후 재시도. |
| `ModuleNotFoundError` | venv가 활성화되지 않았거나 `requirements_clean.txt` 설치 누락. 실행 명령어 확인. |

원본 저자의 프로젝트 개요와 연구 배경은 `README.md`를, 논문은 `PAPER/` 폴더를 참고하세요.
